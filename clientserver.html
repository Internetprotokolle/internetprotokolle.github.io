Die Client-Server-Architektur beschreibt den Austausch von Daten zwischen einem anfragenden Client und einem antworteten Server.

Der Client sendet eine Anfrage an den Server. Diese wird über das Internet zu dem Server geschickt. Dieser verarbeitet die Anfrage und generiert eine Antwort, die an den Server geschickt. Damit sich Server und Client verstehen gibt es Protokolle. Diese definieren, wie die Kommunikation abläuft. Als Dienst wird die Funktion des Servers benannt.

Bekannte Dienste und deren Protokolle

Dienst Protokoll Funktion
Namensauflösung DNS Auflösen von Domain-Namen zu IP-Adressen
World Wide Web HTTP, HTTPS Übertragen von Internetseiten
E-Mail SMTP, POP3, IMAP Übertragen von Internetseiten
Dateiübertragung FTP, FTPS, SFTP Übertragen von Dateien
Konsolenzugriff Telnet, SSH Zugriff auf entfernte Konsole
Internetchat IRC, XMPP Chatten über das Internet
Zeitsynchronisation NTP Synchronisation der Systemzeit mit einem Server

Sicherheit

Standardmäßig sind Verbindungen im Internet unverschlüsselt. Dies ermöglicht Angreifern (Hackern oder auch Staaten) sensible Daten auszulesen und diese im schlimmsten Fall zu modifizieren. Dagegen wurden einige Verfahren entwickelt. Das Weitverbreitetste davon ist TLS. Dieses tauscht bei der Aufbau der Verbindung Schlüssel aus, die mithilfe von Zertifikaten überprüft werden. Sollten die Schüssel akzeptiert werden, wird eine verschlüsselte Verbindung aufgebaut und es ist möglich Daten zu übertragen. Von vielen Protokollen gibt es eine Version in Verbindung mit TLS. Das bekannteste davon ist HTTPS, mit diesem es möglich verschlüsselt Webseiten aufzurufen.
